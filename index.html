APIs

Depending on project needs we build, deploy and keep running escalable REST and GraphQL APIs to support web, mobile, IoT and real-time systems.

Web

  • Traditional Web App: where the application logic reside exclusively in the backend system, we can provide you with a sounded solution leaning on progressive enhancement and ROCA recommendations.

  • Single Page Apps: for rich user interfaces, dashboards and monitoring control panels or when you already have an endpoint serving API request and you need to interact with it through a web browser.

Mobile

  • Native apps for iOS and Android. We use Flutter to share the same codebase for both platforms.

  • Progressive Web Apps when you want to deliver fast not wanting to giving up an amazing user experience while avoiding at the same time the cumbersome approval processes for delivering and update on each platform and, finally, all sharing a large part of the logic used to build the web client.

Deployment

Unless an specific customer requirement force us to lean over another development platform our solutions can usually be deployed, either in bare metal as a single compiled file or, for more complex developments and using container orchestration technologies, in any Kubernetes cloud provider.

Among among other technical features, containers can be seen as a way to avoid vendor lock-in by abstracting the application away from the underlying hardware and platform. Any container image should be able to run on any cloud service (GKE, EKS, Azure, DigitalOcean, etc...)

From Kubernetes sites

Kubernetes is a portable, extensible open-source platform for managing containerized workloads and services, that facilitates both declarative configuration and automation. It has a large, rapidly growing ecosystem.
