How we work

After we receive a formal specification we start building the tests that will ensure compliance during entire development process. Don't have a blueprint for your idea? We can iteratively build it together.

We start building, testing and deploying in our staging servers on regular basis. There you can evaluate each new update. Access to the project dashboard allows you to get updated docs, suggest updates and report issues. We update estimates and trajectory is corrected.

Code review and manual testing is norm on our development strategy. In the same way, automatic tests and static code analysis rule our continuous integration pipelines. Metrics and results are transparently available to you. Always.

Tech Stack

Our Tools

Go Microservices
If exponential demand is anticipated we usually plan the solution as a set of Go microservices deployed to Kubernetes. This ensures us pain-free scalability and absolute portability. GKE, EKS & DigitalOcean Kubernetes are our most used cloud providers but nothing prevents moving our implementation to any other service that supports this technology.
Elixir + Phoenix
When the scope of the application is widely established and high attendance with uniform demand is expected we prefer to develop them using an Elixir-Phoenix combo. Elixir runs on Erlang battle tested BEAM virtual machine; its actor-model concurrency strategy is well know for running low-latency, distributed, and fault-tolerant systems. At the same time, Phoenix LiveView is an exciting new technology that let us build interactive, real-rime apps at almost no cost.
Elm & React
When more complex user interactions are required server side rendering or even LiveView model falls short to implement this need, our primary choice to develop Single Page Applications is Elm. This reactive and purely functional progamming language with its strong static type system and total immutability ensures us runtime-exception-free execution. Working on particular requirements and mainly when future implementations in mobile platforms are foreseen, React is also an appropriate choice for some projects.
